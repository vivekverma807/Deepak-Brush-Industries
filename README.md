# Deepak Brush Industries — Website

A modern, immersive storytelling corporate website for **Deepak Brush Industries**, a prominent manufacturer and supplier of industrial polishing brushes based in Delhi, India.

## 🚀 Overview

This website transforms a traditional industrial brand into a cinematic, narrative-driven experience. It uses advanced CSS and JavaScript to guide users through the company's 40-year legacy, product range, and industry expertise.

## ✨ Key Features

- **Interactive Storytelling**: A chapter-based layout that reveals content as the user scrolls.
- **Modern UI/UX**:
  - **Custom Cursor**: Interactive ring cursor that reacts to hover elements.
  - **Chapter Navigation**: Side-dot navigation to jump between narrative sections.
  - **Animated Title Reveal**: 3D word-by-word reveal for the hero section.
  - **3D Tilt Cards**: Perspective-based hover effects on feature cards.
  - **Smooth Parallax**: Deep background layers that move at different speeds.
- **Responsive Design**: Fully optimized for desktop, tablet, and mobile devices.
- **High Performance**: Lightweight vanilla JS and CSS for fast loading and smooth 60fps animations.

## 🛠️ Technologies Used

- **HTML5**: Semantic structure for SEO and accessibility.
- **CSS3**: Vanilla CSS with modern features like CSS Variables, Grid, Flexbox, and complex animations.
- **JavaScript (ES6+)**: Vanilla JS storytelling engine utilizing Intersection Observer and requestAnimationFrame.

## 📂 Project Structure

- `index.html`: Main storytelling landing page.
- `brushes.html`: Product catalog with category highlights.
- `applications.html`: Detailed use-cases by industry.
- `about.html`: Company history, vision, and milestones.
- `contact.html`: Interactive contact form and location details.
- `css/styles.css`: Core design system and animations.
- `js/main.js`: Interactive functionality and storytelling logic.

## 🏁 Getting Started

Since this is a static website, you can view it directly:

1. Clone the repository.
2. Open `index.html` in any modern web browser.
3. *Recommended*: Run through a local server (like Live Server or `python -m http.server`) for the best experience.
